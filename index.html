When to use Grid?

Grid allows you to create complex layouts to position content in rows and columns.

As well as setting display: grid; there is also display: inline-grid; for the extra level of fun design.

What is Grid?

Grid is an immensely powerful tool used to specify the placement of elements in the document.

Grid and flexbox can work hand in hand for specifying the layout of a document;flexbox for the 1D layout (a single column or row) and grid for the 2D layout of rows and columns together...just as the name suggests...and also like flexbox the source order of the content items does not matter and you can place them in any order in the grid which makes it really simple to rearrange with media queries for responsive design.

How to use Grid?

Using grid will involve setting the grid container and then the placement of the grid items in the container.

Some terms we need to know before we dive into it...

Grid Container
This is the parent or the container for grid where all the grid items will be laid out.
Grid Item
This is a direct descendant or child of the grid-container
Grid Line
These are the row (horizontal) and column (vertical) dividing lines that make up the cells in the grid
Grid Cell
A cell is a single unit of the grid and takes up the space between two adjacant row and column lines
Grid Track
The track takes up the space between two adjacant row lines or column lines...think of it as all the cells along the same row (or column)
Grid Area
The cell is a single unit and the track is a row (or column) of adjacent units but the area takes up the total space within 4 grid lines (must be in a square or rectangular shape)

Display

The first step in creating a grid is to define the element which will be the grid container using display.

div {
display: grid;
} div {
display: inline-grid;
}
Grid Template Columns and Rows

Using a space seperated list of values for the track-size and line-name we can define the columns and rows that make up the grid.

Values for track-size can be...

  • length - 200px
  • percentage - 20%
  • fraction - 1fr
  • auto

A 3 x 3 grid can be created by...

.container {
display: grid;
grid-template-columns: 100px 200px 100px;
grid-template-rows: 50px 15px 25px;
}

This grid can also be created by...

container {
display: grid;
grid-template-columns: 20% 65% 15%;
grid-template-rows: 25% 50% 25%;
}

This grid can even be created by...

.container {
display: grid;
grid-template-columns: 2fr 3fr 1fr;
grid-template-rows: 1fr 1fr 1fr;
}

And another way this grid can be created...

.container {
display: grid;
grid-template-columns: 1fr 200px 2fr;
grid-template-rows: 50px auto 35%;
}

Lastly, we also have the magic of minmax()...

.container {
display: grid;
/*center column will not be smaller than 100px or larger than 500px*/
grid-template-columns: 2fr minmax(100px, 500px) 1fr;
grid-template-rows: minmax(50px, 100px) auto minmax(50px, 100px);
}

If the column or row is defined with only track-size then the grid lines are automatically numbered with positive and negative numbers (postive numbers for the grid lines going from first line to the last line or negative numbers when going from the last grid line to the first grid line) - 3 columns will have 4 lines.

This allows you to target the first cell using the gridline 1 or the last cell using the gridline -1 without knowing the actual size of the grid.

Otherwise you can specifically set names (in []) for the line-name that you create yourself such as [first-line], [row1-start], [col2-end], etc., and the same line can have more than one name [row-1-end row-2-start]

container {
display: grid;
grid-template-columns: [start] 20% [line-1] 65% [line-2] 15% [end];
grid-template-rows: [r1-start] 25% [r1-end r2-start] 50% [r2-end r3-start] 25% [r3-end];
}

Multiple lines can also have the same name...

.container {
display: grid;
grid-template-columns: [col1] 20% [col1] 65% [col1] 15% [col1];
}

and can be referenced by their count...

.item {
grid-column-start: col1 2; /*starts from the second col1 line*/
}

and last of all...we have repeat()!

.container {
display: grid;
grid-template-columns: [col1] 1fr [col1] 1fr [col1] 1fr [col1];
grid-template-rows: 1fr 1fr 1fr;
}

can be written as...

.container {
display: grid;
grid-template-columns: repeat(3, 1fr [col1]);
grid-template-rows: repeat (3 1fr);
}

There is some extra magic we can use with repeat()...auto-fit and auto-fill.

.container {
display: grid;
grid-template-columns: repeat(auto-fit, 200px);
}

Using auto-fit will add extra columns but if no content will size as 0 so no extra blank space shows.

.container {
display: grid;
grid-template-columns: repeat(auto-fill, 200px);
}

Using auto-fill will add extra columns with no content, but shows as blank space.

Grid Template Areas

Using grid-template-areas gives a visual representation of the grid and content. The grid is defined by referencing the grid-area for each cell.

.container {
display: grid;
grid-template-areas:
"header header header"
"nav main none" /*using none means no grid area defined*/
". footer footer"; /*using . means an empty grid cell*/
grid-template-columns: repeat(3 1fr);
grid-template-rows: auto;
}

header {
grid-area: header;
}

nav {
grid-area: nav;
}

main {
grid-area: main;
}

footer {
grid-area: footer;
}

Each row needs to have the same number of cells!!!

When using grid-template-areas the grid-lines are automatically named, for example for the grid area named header the name of the areaâ€™s starting row line and starting column line will be header-start and the name of its last row line and last column line will be header-end (some lines might have multiple names).

Grid Template

The shorthand for for setting grid-template-rows, grid-template-columns, and grid-template-areas is grid-template.

So this...

.container {
display: grid;
grid-template-rows: [row1-start] 25px [row1-end row2-start] 25px [row2-end];
grid-template-columns: auto 50px auto;
grid-template-areas:
"header header header"
"footer footer footer";
}

can instead be written as...

.container {
display: grid;
grid-template:
[row1-start] "header header header" 25px [row1-end]
[row2-start] "footer footer footer" 25px [row2-end]
/ auto 50px auto;
}

The order is grid-template: rows / columns;

However, grid (coming up soon) is used more often than grid-template.

Column Gap & Row Gap

This specifies the size of the grid lines... the space between the grid cells. It is only applied between cells (not the outside edges)

.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
column-gap: 10px;
row-gap: 15px;
}
Grid Gap

You would be correct if you guessed that grid-gap was the shorthand for column-gap and row-gap.

.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-gap: 10px 10px;
}

or...

.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-gap: 10px;
}

If row-gap is not specified it will take the same value as column-gap

Justify Items

To align grid-items along the row axis we use justify-items .

The values that can be set are:

  • start
  • end
  • center
  • stretch
.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-gap: 10px;
justify-items: center;
}
Align Items

To align grid-items along the column axis we use align-items (where justify-items was aligning to the row axis).

The values that can be set are the same as those used with justify-items:

  • start
  • end
  • center
  • stretch
.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-gap: 10px;
justify-items: center;
align-items: center; }
Place Items

The shorthand for justify-items and align-items is...you've guessed it...place-items!

Instead of...

.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-gap: 10px;
justify-items: center;
align-items: center;
}

You can use...

.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-gap: 10px;
place-items: center stretch;
}

The first value is for align-items and the second for justify-items...although if you set only one value both will use it place-items: center;

Justify Content

We use justify-content (and the upcoming align-content) to align the grid along the row (inline) axis when the total size of the grid is smaller than the grid container itself...such as can happen if the grid-items are a fixed size.

The values that can be set are:

  • start
  • end
  • center
  • stretch
  • space-around
  • space-between
  • space-evenly
.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-gap: 10px;
justify-content: center;
}
Align Content

Similar to justify-content we use align-content) to align the grid along the column (block) axis.

The values that can be set are the same as used with justify-content:

  • start
  • end
  • center
  • stretch
  • space-around
  • space-between
  • space-evenly
.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-gap: 10px;
align-content: center;
}
Place Content

The shorthand for justify-content and align-content is...you've guessed coorectly yet again...it is...place-content!

Instead of...

.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-gap: 10px;
justify-content: center;
align-content: center;
}

You can use...

.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-gap: 10px;
place-content: center stretch;
}

The first value is for align-content and the second for justify-content...although if you set only one value both will use it place-content: center;

Grid Auto Columns and Auto Rows

All of the grids so far have been explicit grids in that they are manually defined with set rowns or columns.

We are now going to look at implicit grids. If a grid item is positioned into a column that is not explicitly sized by grid-template-columns , or grid-template-rows implicit grid tracks are created to hold it.

The values we can set for grid-auto-columns or grid-template-rows are the same as used for and grid-template-rows... length (px or rem , etc.), fraction (fr), percentage (%), minmax(), auto...and ones I have mentioned yet...min-content and fit-content!

.container {
display: grid;
grid-auto-columns: minmax(25px, auto);
grid-auto-rows: min-content; }
Grid Auto Flow

So as well as not explicitly defining the columns and rows in the grid (and grid-auto-columns & grid-auto-rows come into play), there is also what happens when grid-items are not explicitly placed which leads us to... auto-placement algorithm which, as the name might kinda give away, is dictating how items are automatically placed in the grid.

The values that can be set are:

  • row
  • column
  • dense
.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
grid-auto-flow: column; grid-gap: 10px;
}
Grid

Now to put it ALL together! grid is the shorthand for setting...grid-template-rows, grid-template-columns, grid-template-areas, grid-auto-rows, grid-auto-columns, and grid-auto-flow all into a single declaration.

So....

.container {
display: grid;
grid-template-columns: 100px auto 100px;
grid-template-rows: 50px 50% 50px;
}

can also be written as....

.container {
display: grid;
grid: 100px auto 100px / 50px 50% 50px;
}

Or....

.container {
display: grid;
grid-auto-flow: row;
grid-template-columns: 100px auto 100px;
}

can also be written as....

.container {
display: grid;
grid: auto-flow / 100px auto 100px;
}

And even....

.container {
display: grid;
grid-auto-flow: row dense;
grid-auto-rows: 100px;
grid-template-columns: 1fr 2fr;
}

can also be written as....

.container {
display: grid;
grid: auto-flow dense 100px / 1fr 2fr;
}

Similar to....

.container {
display: grid;
grid-template-rows: 100px 300px;
grid-auto-flow: column;
grid-auto-columns: 200px;
}

written as....

.container {
display: grid;
grid: 100px 300px / auto-flow 200px;
}

Now are you ready for this....

.container {
display: grid;
grid-template-areas:
"header header header"
"main main main";
grid-auto-flow: column;
grid-auto-columns: 200px;
}

written as....

.container {
display: grid;
grid: [row1-start] "header header header" 1fr [row1-end] [row2-start] "footer footer footer" 25px [row2-end] / auto 50px auto;
}
Grid Items

Now that we have out grid container created we wil look a bit closer at the grid items (grid children) we will be placing in the grid.

Properties for the grid items...

  • grid-column-start
  • grid-column-end
  • grid-row-start
  • grid-row-end
  • grid-column
  • grid-row
  • grid-area
  • justify-self
  • align-self
  • place-self

It will be a 3 x 3 grid used for examples with named lines as follows:

"top-left ... top-center-left ... top-center-right ... top-right"

"center-left ... center-center-left ... center-center-right ... center-right"

"bottom-left ... bottom-center-left ... bottom-center-right ... bottom-right"

Grid Column Start & End and Row Start & End

We use these properties when we want to place a grid-item in the grid by choosing the column or row start line and the column or row end line.

The values that can be set are:

  • line - number
  • line - name
  • span - number
  • spam - name
  • auto

There are a few ways we can do this...starting with the line numbers

.item {
grid-column-start: 1;
grid-column-end: 4;
}

or the negative line numbers...

.item {
grid-column-start: -1;
grid-column-end: -4;
}

or the line numbers...with span

.item {
grid-column-start: 1;
grid-column-end: span 3;
}

or the line names...

.item {
grid-column-start: top-left;
grid-column-end: span bottom-left;
}

If you use auto it sets the item to use an automatic span or placement (default span is 1).

.item {
grid-column-start: auto;
}

And for the other direction...

There are a few ways we can do this...starting with the line numbers

.item {
grid-column-start: -1;
grid-row-start: 1;
grid-row-end: 4;
}

or the negative line numbers...

.item {
grid-column-start: -1;
grid-row-start: -4;
grid-row-end: -1;
}

or the line numbers...with span

.item {
grid-column-start: -1;
grid-row-start: 1;
grid-row-end: span 3;
}

or the line names...

.item {
grid-column-start: top-right;
grid-row-start: top-right;
grid-row-end: span bottom-right;
}

If the column-end or row-end is not specified the item will span 1 track.

Items can overlap too (use z-index for stacking order).

Grid Column and Row

Guess what....there is a shorthand for grid-column-start & grid-column-end and grid-row-start & grid-row-end which is....you guessed it...grid-column and grid-row!!

So...

.item {
grid-column-start: 1;
grid-column-end: span 3;
grid-row-start: 1;
grid-row-end: 4;
}

can be written as...

.item {
grid-column: 1 / span 3;
grid-row: 1 / 4;
}
Grid Area

There are two ways that grid-area can be used:

  1. An even shorter shorthand for grid-column-start & grid-column-end and grid-row-start & grid-row-end
  2. To give an item a name so it can be used hand in hand with named sections of the grid created with grid-template-areas

So using short-shorthand means that...

.item {
grid-column-start: 1;
grid-column-end: span 3;
grid-row-start: 1;
grid-row-end: 4;
}

instead of being written as...

.item {
grid-column: 1 / span 3;
grid-row: 1 / 4;
}

is written as...

.item {
/* row-start / column-start / row-end / column-end */
grid-area: 1 / 1 / 4 / span 3;
}

BUT...my prefered method would be using it this way...

.container {
display: grid;
grid-template-areas:
"header header header"
"nav main main"
"footer footer footer";
grid-template-columns: repeat(3 1fr);
grid-template-rows: auto;
}

header {
grid-area: header;
}

nav {
grid-area: nav;
}

main {
grid-area: main;
}

footer {
grid-area: footer;
}
Justify Self

This applies only the grid item in a single cell and is used to align along the row (inline) axis

The values that can be set are:

  • start
  • end
  • center
  • stretch
.item {
justify-self: center;
}
Align Self

Using justify-self aligns along the row (inline) axis and align-self aligns along the column (block) axis.

The values are the same as those available for justify-self:

  • start
  • end
  • center
  • stretch
.item {
justify-self: center;
align-self: center;
}
Place Self

You'll never have guessed this but place-self is the shorthand for justify-self and align-self!

So we can write...

.item {
align-self: center;
justify-self: start;
}

You can use...

.item {
place-self: center start;
}

The first value is for align-self and the second for justify-self...although if you set only one value both will use it place-self: center;
